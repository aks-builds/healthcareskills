## Important reminders for agents

- These skills must be **safe to use clinically**. Never invent dosing, diagnoses, billing codes, or regulatory citations. When uncertain, tell the user "verify against the current source" rather than guess.
- These skills are **not a substitute for clinical, legal, or regulatory advice**. They help engineers, informaticists, and product teams build correct healthcare software — not provide medical advice to patients.
- Treat PHI (Protected Health Information) with care in every example. Default to synthetic data in code samples; never paste real patient identifiers.
